技术栈与 TUI 布局选择

既然由我来做主，我推荐使用 Rust (Aya)。它对 eBPF 的支持非常前卫，能保证内存安全，且编译出的二进制文件极小，非常适合这种底层系统工具。

1. 核心技术栈

  • 开发语言： Rust (强类型、高性能、内存安全)
  • eBPF 框架： Aya-rs (纯 Rust 实现的 eBPF 库，支持 CO-RE)
  • TUI 框架： Ratatui (目前 Rust 最流行、功能最强大的终端 UI 库)

2. TUI 界面布局方案

我们需要一个直观的界面来管理屏蔽列表。你更喜欢哪种风格？

A

仪表盘风格 (Dashboard)

特点： 左侧导航栏（状态、列表、设置），右侧详细信息。类似现代应用的布局。

B

极简列表风格 (Minimalist)

特点： 整个屏幕就是一个大列表，下方有简单的快捷键提示 (Add/Delete/Start)。

C

经典双栏管理 (Two-Pane)

特点： 类似 Total Commander，左侧是预设列表，右侧是已启用的屏蔽，方便快速切换。

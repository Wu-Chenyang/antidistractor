挑战：如何绕过代理软件（如 Clash）？

代理软件（尤其是开启 TUN 模式或 Fake-IP 时）会接管系统的 DNS 和流量，导致传统的 /etc/hosts 方案失效。

流量路径分析
浏览器 / 应用
(请求 google.com)
→
Clash (TUN/Fake-IP)
拦截请求，返回虚拟 IP
→
互联网
(加密隧道 / 直接连接)

传统的 /etc/hosts 修改在这里往往被完全跳过。

针对这一情况的 3 种技术方向：

A

方案一：Clash API 联动

原理： 通过 Clash 的外部控制 API 动态注入 REJECT 规则。
优点： 最简单、不影响性能、支持 Fake-IP。
缺点： 强依赖于 Clash，换个代理软件就失效了。

B

方案二：eBPF 监控 (推荐)

原理： 在内核层挂钩，通过检测 TLS 握手中的 SNI (域名) 来直接丢包。
优点： 极其难以绕过，与代理软件无关，系统级生效。
缺点： 开发难度大，需要较新的内核版本，且需要 root 权限。

C

方案三：Nftables 标记丢包

原理： 配合 Clash 的 fake-ip-filter，强制某些域名走真实 IP，然后用 Nftables 封锁。
优点： 比较标准、性能好。
缺点： 配置比较碎，用户需要手动修改 Clash 配置来配合。

考虑到你长期使用 Clash，你更倾向于哪种深度？

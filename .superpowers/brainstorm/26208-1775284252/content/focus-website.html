第一阶段：专注网页屏蔽

既然我们决定第一版只屏蔽网站，我们需要选择一种最有效且难以绕过的方法。

实现方式建议

在 Linux 中，有几种常见的方法可以实现系统级的网页屏蔽：

A

修改 /etc/hosts

优点： 简单、生效快、系统级生效。
缺点： 需要 root 权限，无法处理 HTTPS SNI 级别的精细控制。

B

Iptables / Nftables 防火墙

优点： 非常强大，可以根据 IP 或端口屏蔽。
缺点： 配置复杂，难以动态更新庞大的域名列表。

C

本地 DNS 劫持

优点： 灵活，可以对特定域名进行拦截。
缺点： 设置复杂（需配置 systemd-resolved 等），容易被自定义 DNS 绕过。
